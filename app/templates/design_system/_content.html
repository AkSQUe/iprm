Interactions

Анімації та переходи

Easing Curves (Apple-specific)

--ease-reveal: cubic-bezier(0.28, 0.11, 0.32, 1) -- Reveal / fade-in
--ease-interact: cubic-bezier(0.25, 0.46, 0.45, 0.94) -- Interaction (hover, press)
--ease-spring: cubic-bezier(0.32, 0.08, 0.24, 1) -- Spring-like bounce

Apple NEVER uses generic ease or linear. Always specific cubic-bezier with deceleration at the end.

Scroll Reveal (CSS-first, JS fallback)

CSS Scroll-Driven Animations (primary):
.apple-reveal { animation: enter-view linear both; animation-timeline: view(); animation-range: entry 0% entry 40%; }
opacity: 0, translateY(24px) -> opacity: 1, translateY(0)

JS Fallback (IntersectionObserver):
.apple-reveal -- threshold: 0.12, rootMargin: '0px 0px -40px 0px'
Duration: 0.8s cubic-bezier(0.28, 0.11, 0.32, 1), one-time (unobserve after)

Default (no animation): .apple-reveal { opacity: 1; transform: none; } -- visible for reduced-motion and fallback

Hero fadeUp

@keyframes appleHeroFadeUp { to { opacity: 1; transform: translateY(0); } }
Start: opacity: 0, translateY(20-30px)
Badge: delay 0s / h1: 0.15s / subtitle: 0.3s / actions: 0.45s
Duration: 0.8s cubic-bezier(0.28, 0.11, 0.32, 1)

Hover ефекти

Cards: translateY(-4px) + box-shadow 0 12px 40px rgba(0,0,0,0.1) -- 0.3s
Bento: scale(1.01) -- 0.3s
Buttons: scale(1.02) translateY(-1px) -- 0.15s
Links: color transition -- 0.2s
Easing: cubic-bezier(0.25, 0.46, 0.45, 0.94) for all interactions

Dropdown staggered fade-in

.dropdown-item:nth-child(1) { transition-delay: 0.02s; }
.dropdown-item:nth-child(2) { transition-delay: 0.05s; }
.dropdown-item:nth-child(3) { transition-delay: 0.08s; }
.dropdown-item:nth-child(4) { transition-delay: 0.11s; }

4 правила Apple-руху

1. Duration < 300ms для UI-відповідей -- довше відчувається повільним
2. Ease curves, ніколи linear -- cubic-bezier з уповільненням в кінці
3. Рух має сенс -- кожна анімація комунікує (зв'язок, стан, ієрархію)
4. Субтильність > спектакль -- найкращі анімації ті, що користувач відчуває, але не може описати

prefers-reduced-motion: анімації ВИДАЛЯЮТЬСЯ, не сповільнюються. Сповільнені анімації можуть викликати більше нудоти у вестибулярно-чутливих користувачів.

Accessibility

Focus indicators (Apple pattern)

:focus-visible { outline: 2px solid var(--accent); outline-offset: 3px; }

prefers-reduced-motion

@media (prefers-reduced-motion: reduce) { .apple-reveal { transition: none; opacity: 1; transform: none; animation: none; } }
Apple rule: REMOVE animations, never slow them down.

prefers-reduced-transparency

@media (prefers-reduced-transparency: reduce) { .iprm-header, .cookie-banner, .dropdown { background: rgba(255,255,255,0.95); backdrop-filter: none; } }

prefers-contrast: high

@media (prefers-contrast: more) { :root { --black: #000; --gray: #000; --section-bg: #fff; } .glass-panel { backdrop-filter: none; border: 2px solid #000; } }

Optimization

Performance (Apple patterns)

CSS Containment

.iprm-section { content-visibility: auto; contain-intrinsic-size: auto 600px; }
Below-fold sections not rendered until needed -- significant paint time reduction.

Font loading

<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preload" href="/static/fonts/inter-400.woff2" as="font" type="font/woff2" crossorigin>
font-display: swap for Inter.

Images

<picture> with WebP + JPEG fallback
Above-fold: loading="eager" fetchpriority="high"
Below-fold: loading="lazy"
Always explicit width and height to prevent CLS.

will-change

.card:hover { will-change: transform; }
Only during animation, never permanently. Permanent will-change consumes GPU memory.

Apple pattern: performance is invisible. Users should never notice optimization -- they should notice speed.

Progressive Enhancement

Modern CSS

Container Queries

Components adapt to container, not viewport:
.card-container { container-type: inline-size; }
@container (min-width: 400px) { .card-body { grid-template-columns: 1fr 1fr; } }

CSS Scroll-Driven Animations

Replace JavaScript scroll listeners for reveal animations:
.apple-reveal { animation: enter-view linear both; animation-timeline: view(); animation-range: entry 0% entry 40%; }

View Transitions (page navigation)

@view-transition { navigation: auto; }
::view-transition-old(root) { animation: fade-out 0.25s ease-out; }
::view-transition-new(root) { animation: fade-in 0.25s ease-in; }

color-mix() for hover states

.btn:hover { background: color-mix(in oklch, var(--accent), black 15%); }
Programmatic darkening without separate hover variables.

All modern CSS features use progressive enhancement -- fallbacks always provided for older browsers.
